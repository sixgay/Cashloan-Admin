建议

  • src/api的api模块有点过度拆分，可以将admin.js, loan.js, login.js, menu.js 全部整合在一个封装的方法里面
  • src/lang的多语言模块，文件拆分有点过度，然后语言翻译不应该全部都由前端来翻译，这会导致表格过多处理语言转化会有卡顿的现象， 比如接口返回数据和表单的选择项，可有后端通过key，value的形式进行翻译赋值，可参照星合的模式。

# 案件模块

注解

案件模块目前有二十多个模块，通过申请审批配置管理来进行模块对应不同角色的配置，当用户登录时，会获取配置的模块，存储localstroage中。等到访问案件详情页时会根据当前角色匹配对应的模块，使用component进行模块化动态加载。

  • 使用方式:
// tab 用于生成接口变量
// name 模块名称
<template v-for="item in common_bottom_info_list">
  <component :key="item" :name="item" :tab="common_bottom_info" :is="item"></component>
</template>

# env.process环境变量

提示

第一次拉取代码，执行npm run copyenv 指令来拷贝生成local.env.js , 由于config/index.js引入了local的配置， 不执行会导致npm run dev 会报错。

├─ src
│  ├─ config 
│  │  ├─ dev.env.js 上传到git的开发环境
│  │  ├─ local.env.js 忽略上传到git的开发环境(主要方便用于后台人员本地运行项目调试)
│  │  ├─ prod.env.js 生产环境
  • 环境变量的参数说明: 例dev.env.js
'use strict'
const merge = require('webpack-merge')
const prodEnv = require('./prod.env')

// proxyTable 用于本地开发解决跨域问题
// BASE_API 用于线上环境url，本地开发获取登录页图形验证码
// UPLOAD_API 用于上传的api
// SOCKET_API 用于websocket的api
// NODE_ENV 当前环境的标识
module.exports = merge(prodEnv, {
  proxyTable: 'http://192.168.1.199:8100',
  BASE_API: '"http://cashloan_api.leying.me"',
  UPLOAD_API: '"/api/admin/advertise/upfile"',
  SOCKET_API: '"192.168.1.199:8100"',
  NODE_ENV: '"development"',
})

# vuex

  • vuex全局状态: 例getters.js
const getters = {
  sidebar: state => state.app.sidebar, // 当前菜单导航栏是否打开
  language: state => state.app.language, // 当前语言(中文/英文/印尼文)
  device: state => state.app.device, //  当前设备(桌面/移动)，根据屏幕大小确定
  token: state => state.user.token, // 当前用户token
  targetId: state => state.user.targetId, // 拖拽菜单的目标节点
  whereInsert: state => state.user.whereInsert, // 拖拽菜单到目标节点的位置(上/中/下)
  nav: state => state.user.nav, // 导航菜单数据
  visitedViews: state => state.tagsView.visitedViews, // 记录导航标签的属性
  cachedViews: state => state.tagsView.cachedViews, // 记录导航标签的name
  task_status: state => state.user.task_status, // 用于检测案件详情的审批任务是否已全部完成，才能进行贷款信息的通过
  orderStatus: state => state.user.orderStatus // 用于检测当前案件详情的订单状态
}
export default getters
