# 组件化

本文讲解Cashoan Admin项目中主要使用到组件

# BackToTop

功能

滚动到底部和回滚到顶部，主要用于案件详情页

# 参数

参数名 介绍 必选 值类型 可选值 默认值
backPosition 返回时距离顶部的高度 非 Number 300
visibilityHeight 滚动到多大的高度才能看到回到顶部的图标 非 Number 400
customStyle 回到顶部图标的位置 非 Object {right: '50px',bottom:'50px','z-index': '9'}
customBottomStyle 滚动到底部的图标位置 非 Object {right: '50px',top:'200px','z-index': '9'}

# SearchTablePagination beta

功能

将表单、表格、分页一体化，这里是配置最多的组件，基本上所有管理页面都会使用到

  • 属性:
参数 说明 类型 可选值 默认值
data 数据集合(数据结构已和后端严格定义好restful风格，已在内部统一做了处理) array - -
url 后端数据接口(api统一存放在/src/api/index) string - -
method 接口请求方式 string get, post, delete, put get
params 搜索参数 object - -
form-options form表单设置(见下方Form Option 属性) object - -
data-handler 数组数据的 map 处理函数 function - -
columns table column配置对象的数组。具体配置见下面Table column 属性 array - -
show-pagination 是否显示分页组件，如设为false，查询时不传分页参数 boolean true, false true
page-sizes 每页显示个数的控件选项 array - [20, 50, 100]
pagination-layout 分页控件的结构，每个类型用逗号分隔 string sizes, prev, pager, next, jumper, ->, total, slot total, prev, pager, next, jumper, sizes
page-index-key 参数：页码数 的 key 值 string - pageIndex
page-size-key 参数：每页展示个数 的 key 值 string - pageSize

注：更多属性，请参考 Element UI Table 组件的文档。

支持的Element UI Table属性有：stripe / border / height / max-height / fit / show-header / highlight-current-row / current-row-key / row-class-name / row-style / row-key / empty-text / default-expand-all / expand-row-keys / default-sort / tooltip-effect / show-summary / sum-text / summary-method / size。

# dragTreeTable beta

功能

可以拖拽排序的树形表格，主要用于菜单管理

  • 使用方式:
  <dragTreeTable 
    :data="treeDataLeft" 
    :data-other="treeDataRight" 
    :on-drag="onTreeDataChangeLeft" 
    :isdraggable="true"/>
  • 属性:
参数 说明 类型 可选值 默认值
data 数据集合data-option 属性 array - -
data-other 另外一组树形表格数据集合 array - -
isdraggable 是否可拖拽 boolean - true
on-drag 当拖拽改变数据时，触发的回调函数 function

# data-option属性

参数 说明 类型 可选值 默认值
columns 数据集合colums 属性 array - -
children 树形结构 array - -
custom_field 数据源配置custom_field 属性 boolean - true

参数 说明 类型 可选值 默认值
type selection会显示折叠图标，actions指操作栏, checkbox支持多选全选 string 'selection', 'actions', 'checkbox' -
title 表格标题 string - -
width 单元格宽度 number - -
align 单元格对齐方式，默认局左对齐 string left,center,right
formatter 自定义单元格显示内容,参数为当前行数据 Function
field 单元格内容取值使用 String

# custom_field属性

参数 说明 类型 可选值 默认值
id 唯一标志, string -
parent_id 父节点ID string - -
order 排序,0开始,onDrag后，order会重置 Number - -
lists 子节点 string

优化建议

该组件初始原型参考vue-drag-tree-table，对其进行改装成适合当前项目的组件。

  • 待优化的点:
    • 目前只能进行组件内树形拖拽和两个组件相互拖拽，可提升成多个树形组件相互拖拽。
    • 性能的上优化,目前当完成拖拽的节点后，重新生成树形数据时，会先从另外的组件开始进行递归，然后再去自己的组件内进行计算，导致两个树形的结构都重新生成了。但我只在当前的树形结构进行拖拽，其实是只需重新生成当前的树形结构就好了，没有必要两个树形都重新生成

# 主题配置

一个 VuePress 主题应该负责整个网站的布局和交互细节。在 VuePress 中，目前自带了一个默认的主题（正是你现在所看到的），它是为技术文档而设计的。同时，默认主题提供了一些选项，让你可以去自定义导航栏（navbar）、 侧边栏（sidebar）和 首页（homepage） 等，详情请参见 默认主题 。

如果你想开发一个自定义主题，可以参考 自定义主题。

# 应用级别的配置

由于 VuePress 是一个标准的 Vue 应用，你可以通过创建一个 .vuepress/enhanceApp.js 文件来做一些应用级别的配置，当该文件存在的时候，会被导入到应用内部。enhanceApp.js 应该 export default 一个钩子函数，并接受一个包含了一些应用级别属性的对象作为参数。你可以使用这个钩子来安装一些附加的 Vue 插件、注册全局组件，或者增加额外的路由钩子等：

export default ({
  Vue, // VuePress 正在使用的 Vue 构造函数
  options, // 附加到根实例的一些选项
  router, // 当前应用的路由实例
  siteData // 站点元数据
}) => {
  // ...做一些其他的应用级别的优化
}
